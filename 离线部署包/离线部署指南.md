## 前提条件

| 项目 | 要求 |
|---|---|
| 镜像仓库 | 镜像已推送至 `registry.epoint.com.cn/image-rotation-api:2.0.0` |
| 构建服务器 | 可访问内网镜像仓库，安装了 Docker |
| 生产服务器 | 网络隔离区，无外网访问，安装了 Docker |
| 镜像传输介质 | U盘 / 内网文件服务器 |

> **镜像已在内网仓库，无需重新构建。** 以下步骤仅导出离线包并部署到隔离区服务器。

---

## 1. 导出离线镜像包（构建服务器上执行）

### 1.1 登录内网镜像仓库

```bash
docker login registry.epoint.com.cn
```

### 1.2 拉取镜像

```bash
docker pull registry.epoint.com.cn/image-rotation-api:2.0.0
```

### 1.3 导出为离线包

```bash
mkdir -p /opt/images
docker save registry.epoint.com.cn/image-rotation-api:2.0.0 \
    -o /opt/images/image-rotation-api-2.0.0.tar

gzip -c /opt/images/image-rotation-api-2.0.0.tar \
    > /opt/images/image-rotation-api-2.0.0.tar.gz
```

文件大小约 **600–800 MB**（压缩后）。

### 1.4 拷贝到 U 盘

```bash
# U 盘挂载到 /mnt/usb
cp /opt/images/image-rotation-api-2.0.0.tar.gz /mnt/usb/
```

---

## 2. 导入生产服务器

### 2.1 拷贝镜像包到服务器

```bash
# U 盘插入生产服务器，假设挂载到 /mnt/usb
cp /mnt/usb/image-rotation-api-2.0.0.tar.gz /opt/images/
```

### 2.2 导入镜像

```bash
gunzip -c /opt/images/image-rotation-api-2.0.0.tar.gz | docker load

# 确认导入成功
docker images image-rotation-api:2.0.0
```

---

## 3. 启动服务

### 3.1 基础启动

```bash
docker run -d \
    --name image-rotation-api \
    --restart unless-stopped \
    -p 8000:8000 \
    -v /opt/logs/image-rotation-api:/app/logs \
    image-rotation-api:2.0.0
```

### 3.2 生产级启动（推荐）

```bash
docker run -d \
    --name image-rotation-api \
    --restart unless-stopped \
    --log-opt max-size=100m \
    --log-opt max-file=5 \
    -p 8000:8000 \
    -e LOG_LEVEL=INFO \
    -e WORKERS=2 \
    -v /opt/logs/image-rotation-api:/app/logs \
    -v /opt/data/input:/data/input \
    -v /opt/data/output:/data/output \
    --health-cmd="python -c 'import httpx; httpx.get(\"http://localhost:8000/health\", timeout=5).raise_for_status()'" \
    --health-interval=30s \
    --health-timeout=10s \
    --health-retries=3 \
    --health-start-period=20s \
    image-rotation-api:2.0.0
```

### 3.3 使用 docker-compose（推荐方式）

创建 `/opt/image-rotation-api/docker-compose.yml`：

```yaml
services:
  api:
    image: image-rotation-api:2.0.0
    container_name: image-rotation-api
    restart: unless-stopped
    ports:
      - "8000:8000"
    environment:
      - LOG_LEVEL=INFO
      - WORKERS=2
    volumes:
      - /opt/logs/image-rotation-api:/app/logs
    healthcheck:
      test: ["CMD", "python", "-c", "import httpx; httpx.get('http://localhost:8000/health', timeout=5).raise_for_status()"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 20s

networks:
  default:
    driver: bridge
```

启动：

```bash
cd /opt/image-rotation-api
docker compose up -d
```

## 4. 验证

```bash
# 健康检查
curl http://localhost:8000/health | python -m json.tool

# 预期输出：
# {
#     "status": "healthy",
#     "pid": <数字>,
#     "cpu_count": <数字>,
#     "model_loaded": true,
#     "uptime": "Xs"
# }

# 测试一张正向图片（应返回 0°）
curl -s -X POST http://localhost:8000/v1/image/auto-orient \
  -F "file=@/path/to/upright.jpg;type=image/jpeg" | \
  python -c "import json,sys; d=json.load(sys.stdin); print('旋转角:', d['metadata']['total_correction_deg'], '°')"

# 预期：旋转角: 0.0 °

# 测试一张倒置图片
curl -s -X POST http://localhost:8000/v1/image/auto-orient \
  -F "file=@/path/to/rotated.jpg;type=image/jpeg" | \
  python -c "import json,sys; d=json.load(sys.stdin); print('旋转角:', d['metadata']['total_correction_deg'], '°')"

# 预期：旋转角: 90.0 ° 或 180.0 ° 或 270.0 °
```

**API 文档**（浏览器打开）：

```
http://<服务器IP>:8000/docs
```

**测试页面**：

```
http://<服务器IP>:8000/test/
```

## 5. 更新版本

### 5.1 在构建服务器上导出新版本

```bash
docker pull registry.epoint.com.cn/image-rotation-api:<新版本>
docker save registry.epoint.com.cn/image-rotation-api:<新版本> \
    | gzip -c > /opt/images/image-rotation-api-<新版本>.tar.gz
```

拷贝到 U 盘，再到生产服务器执行导入。

### 5.2 在生产服务器上热更新

```bash
# 导入新镜像
gunzip -c /path/to/new-image.tar.gz | docker load

# 停止旧容器
docker stop image-rotation-api
docker rm image-rotation-api

# 重新启动（使用新版本 tag）
docker run -d \
    --name image-rotation-api \
    --restart unless-stopped \
    -p 8000:8000 \
    -v /opt/logs/image-rotation-api:/app/logs \
    image-rotation-api:<新版本>
```

## 附录：内网（AISP）部署（Dockerfile-aisp）

如需将 Python API 集成到点政系统的 Tomcat 镜像中，使用 `Dockerfile-aisp`。

### A. 构建

```bash
# 1. 登录内网仓库，拉取基础镜像
docker login registry.epoint.com.cn
docker pull registry.epoint.com.cn/base/epoint_centos_tomcat8_openjdk8:v0.3

# 2. 构建
docker build -f Dockerfile-aisp -t registry.epoint.com.cn/image-rotation-api:2.0.0 .

# 3. 导出
docker save registry.epoint.com.cn/image-rotation-api:2.0.0 \
    | gzip -c > image-rotation-api-2.0.0-aisp.tar.gz
```

### B. 启动

```bash
docker run -d \
    --name image-rotation-api \
    --restart unless-stopped \
    -p 8000:8000 \
    -p 8080:8080 \
    -v /opt/logs/image-rotation-api:/app/logs \
    -v /opt/logs/tomcat:/usr/local/tomcat/logs \
    image-rotation-api:2.0.0
```

### C. 验证

```bash
curl http://localhost:8000/health      # Python API
curl http://localhost:8000/test/        # 测试页面
curl http://localhost:8080/            # Tomcat
```

### D. 架构

```
Supervisord（PID 1）
  ├─ tomcat  port 8080  — 点政服务
  └─ api     port 8000  — 图片旋转 API
```

### E. 构建前提

| 项目 | 要求 |
|---|---|
| Docker | 已安装 |
| 内网访问 | 能访问 `registry.epoint.com.cn` |
| App/ 目录 | 可选；有则部署到 Tomcat，无则只运行 Python API |
